# Phase 6 — Evaluation & Release ⬜

**Goal.** Close the loop like a research project: measure it, make it
reproducible, write it up, and ship v1.0.

**Status:** ⬜ Planned · **Depends on:** Phase 5

---

## Sub-phases

### 6.1 Formal evaluation + user study
- Objective metrics (transcription, harmonization, playability) + a small
  blind listening / usability study.
- **DoD:** reported metrics + survey results.

### 6.2 Reproducibility
- Pinned configs, seeds, model cards, dataset manifests; one-command reproduction
  of the trained model.
- **DoD:** a fresh machine reproduces the model and figures.

### 6.3 Write-up
- Technical report / blog post / paper draft describing method and results;
  diagrams from `docs/ARCHITECTURE.md`.
- **DoD:** a shareable document.

### 6.4 Release v1.0
- `CHANGELOG.md`, `CITATION.cff`, semantic-version tag, release notes, archived
  demo + weights.
- **DoD:** tagged `v1.0.0`.

## Definition of Done (phase)
- [ ] Evaluation report + user study.
- [ ] Fully reproducible.
- [ ] `v1.0.0` released with citation.
